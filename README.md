# image-sense

使用**本地 OpenAI 兼容 VLM（带视觉能力）批量识别当前目录下的图片（jpg/png），给出**中文文件名改名建议，并输出可直接执行的 `mv` 命令。

## 1. 前置：配置本地模型

工具使用本地 vlm 模型，会优先读取环境变量；如未设置，会尝试从 `~/.zshrc` 中解析以下变量：

```bash
export LOCAL_API_KEY="sk-xxxxxxxxxxxxxxxxxxxxx"
export LOCAL_BASE_URL="http://127.0.0.1:8086/v1"
export LOCAL_MODEL="gemma-4-e2b-it-4bit"
```

> 说明：这是 OpenAI 兼容接口（`/v1`），并且模型具备视觉能力（VLM）。

## 2. 安装

```bash
pnpm add -g image-sense
```

## 3. 使用

在**包含图片的目录**下执行：

> 提示：工具会先输出执行进度（正在识别哪张图、已完成数量等），待全部识别完成后，再统一输出 `mv` 命令，避免你在控制台里“卡住”不知道发生了什么。

### 3.1 默认规则（不带 -c/-p/-b/-n）

- 规则 1：若图片里只有一个人物 → `姿势_穿着`
- 规则 2：若有两个或更多人物 → `活动`

```bash
image-sense
```

输出示例（每个文件一行）：

```bash
mv "IMG_0001.jpg" "比心_白色运动衫.jpg"
mv "IMG_0002.png" "跑步.png"
```

### 3.2 自定义拼词（出现顺序即词序）

> 除了 `-s` 外，只要出现任意一个文件名参数（`-c/-p/-b/-n`），就会**忽略默认命名规则**，并按参数顺序拼接。

- `-c`：穿着词（cosplay 可用“超人装”，日常用“黑色夹克/白色运动衫”等）
- `-p`：姿势词（如“下蹲”“比心”）
- `-b`：行为词（如“跑步”“园艺活动”）
- `-n`：人数词（如“单人”“双人”“三人”…）
- 连接符：下划线 `_`

例如：

```bash
image-sense -n -b -c
```

可能输出：

```bash
mv "a.jpg" "双人_合影_黑色夹克.jpg"
```

### 3.3 保存到文件

保存到指定文件名：

```bash
image-sense -s "rename.sh"
```

未提供文件名则默认保存为 `rename.sh`：

```bash
image-sense -s
```

### 3.4 自动执行改名（会写 rename.sh 作为日志）

`-r` 会在生成计划后**自动执行改名**，并且**强制写入** `rename.sh`（作为日志）。建议先不带 `-r` 预览，或先用 `-s` 保存检查。

```bash
image-sense -r
```

## 4. 重名处理

如果同一个文件夹下出现**相同的目标文件名**，会自动追加后缀 `_1`、`_2`…（在扩展名前）：

```bash
mv "1.jpg" "跑步.jpg"
mv "2.jpg" "跑步_1.jpg"
```

## 5. 提示

- 输出的 `mv` 命令默认带引号，便于在 macOS / zsh 下直接执行。
- 本地模型输出不可控时，建议先 `-s` 保存到脚本里人工检查后再执行。
